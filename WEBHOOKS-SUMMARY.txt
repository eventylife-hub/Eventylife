CRITICAL DECISIONS MADE
================================================================================

1. IDEMPOTENCE via StripeEvent
   - Create unique constraint on stripeEventId
   - Catch P2002 error → skip processing
   - Prevents any double processing

2. NO THROW IN WEBHOOKS
   - Never throw (Stripe retries infinitely)
   - Always return 200 OK to Stripe
   - Mark event processedAt=null if error
   - CRON monitor detects failed events

3. ATOMIC TRANSACTIONS
   - All updates in $transaction()
   - No race conditions
   - Full rollback on error

4. STATUS GUARDS
   - updateMany with where status conditions
   - Prevents overwriting SUCCEEDED → FAILED
   - Handles concurrent duplicates

5. ADMIN ALERTING
   - CRITICAL events → immediate email
   - ConfigService for ADMIN_ALERT_EMAIL
   - Async (non-blocking webhook)
